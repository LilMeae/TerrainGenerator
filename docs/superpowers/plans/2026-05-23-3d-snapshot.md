# 3D Terrain Snapshot — Save as PNG Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extend the existing "Save as PNG" button so it saves the currently displayed 3D view at 1920×1080 when the 3D view is active, while leaving the 2D save path unchanged.

**Architecture:** Add a `snapshot(width, height)` method on `TerrainView3D` that resizes the SubScene to the target dimensions, calls `SubScene.snapshot(...)` synchronously on the JavaFX Application Thread, restores the original dimensions, and returns a `BufferedImage`. `MainController.onSave()` branches on `ui.getView3DButton().isSelected()` and routes to either the new 3D path or the existing 2D path. Both paths converge on `ImageIO.write(img, "png", file)`.

**Tech Stack:** Java 17+, JavaFX (controls + swing), Maven (`javafx-maven-plugin`), `javax.imageio.ImageIO`, `javafx.embed.swing.SwingFXUtils`.

**Spec reference:** `docs/superpowers/specs/2026-05-23-3d-snapshot-design.md`

**Note on testing:** This project has no unit-test harness configured (`pom.xml` declares no test plugin or dependency; there is no `src/test/java`). Verification is therefore manual — launch the JavaFX app via `mvn javafx:run` and exercise the UI per the steps in Task 3. Do not add JUnit or any test runner as part of this work; it is out of scope.

At the bottom of class `TerrainView3D`, after the existing `resetCamera()` method (currently ending at line 183 with the closing `}` of `resetCamera`) and before the class's closing brace (line 184), add:

```java
    /**
     * Renders the current 3D scene to a BufferedImage at the requested pixel
     * dimensions. The SubScene is temporarily resized to (width, height) for
     * the snapshot and restored immediately afterward; because this all runs
     * on the JavaFX Application Thread inside a single event handler, the
     * resize is not visible to the user.
     *
     * Returns null if no terrain mesh has been built yet.
     */
    public BufferedImage snapshot(int width, int height) {
        if (terrainGroup.getChildren().isEmpty()) {
            return null;
        }

        double originalWidth  = subScene.getWidth();
        double originalHeight = subScene.getHeight();

        subScene.setWidth(width);
        subScene.setHeight(height);

        SnapshotParameters params = new SnapshotParameters();
        params.setFill(Color.rgb(180, 195, 210));

        WritableImage fxImage = new WritableImage(width, height);
        subScene.snapshot(params, fxImage);

        subScene.setWidth(originalWidth);
        subScene.setHeight(originalHeight);

        return SwingFXUtils.fromFXImage(fxImage, null);
    }
```

Notes:
- `Color` is already imported (`javafx.scene.paint.Color`) and used by the existing `subScene.setFill(Color.rgb(180, 195, 210))` call at line 61 — the snapshot fill matches that value deliberately.
- Do not call `params.setCamera(...)`. `SubScene` already owns its camera; `snapshot()` uses it automatically.
- Do not change anything else in the file.

Replace it with the following:

```java
    private void onSave(){
        FileChooser fileChooser = new FileChooser();
        fileChooser.setTitle("Save Terrain Map");
        fileChooser.getExtensionFilters().add(new ExtensionFilter("PNG Image", "*.png"));
        File file = fileChooser.showSaveDialog(null);
        if (file == null) return;

        BufferedImage img;
        if (ui.getView3DButton().isSelected()) {
            img = view3D.snapshot(SNAPSHOT_WIDTH, SNAPSHOT_HEIGHT);
        } else {
            javafx.scene.image.Image fxImage = ui.getMapView().getImage();
            if (fxImage == null) return;
            img = SwingFXUtils.fromFXImage(fxImage, null);
        }
        if (img == null) return;

        try {
            ImageIO.write(img, "png", file);
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
```

Notes:
- The existing imports (`BufferedImage`, `SwingFXUtils`, `ImageIO`, `FileChooser`, `ExtensionFilter`, `File`, `IOException`) all stay — no new imports needed in this file.
- `ui.getView3DButton()` already exists on `UIBuilder` and returns the radio/toggle button driving the existing `onViewToggle()` listener; do not add a new accessor.
- `view3D` is the already-initialized `TerrainView3D` field on this class.
